All local clients connect to the same endpoint:

```text
http://127.0.0.1:9680/mcp/
```

Use this local setup flow:

1. Install the plug-in and dependencies with `python3 install.py`
2. Link `Codex` or `Claude Code`
3. Start the server in Glyphs with **Edit → Start MCP Server**
4. Verify with `list_open_fonts`

No proxy is needed for the supported local clients.

:::tip Startup order
If your client cannot connect, quit it, start Glyphs fresh, run **Edit → Start MCP Server**, then relaunch or reload the client.
:::

<Tabs>
  <TabItem value="codex" label="Codex" default>
    <p>Add the local MCP server with the Codex CLI:</p>

    <CodeBlock language="bash">{`codex mcp add glyphs-mcp-server --url http://127.0.0.1:9680/mcp/
codex mcp list`}</CodeBlock>

    <p>
      Codex can also read the same configuration directly from <code>~/.codex/config.toml</code>:
    </p>

    <CodeBlock language="toml">{`[mcp_servers.glyphs-mcp-server]
url = "http://127.0.0.1:9680/mcp/"
enabled = true
startup_timeout_sec = 30
tool_timeout_sec = 120`}</CodeBlock>

    <p>Verify with a read-only prompt:</p>

    <CodeBlock language="text">{`Call list_open_fonts and tell me how many fonts are open.
If you see an error, quote it verbatim.`}</CodeBlock>
  </TabItem>

  <TabItem value="claude_code" label="Claude Code">
    <p>Add the local MCP server with the Claude CLI:</p>

    <CodeBlock language="bash">{`claude mcp add --scope user --transport http glyphs-mcp http://127.0.0.1:9680/mcp/
claude mcp list`}</CodeBlock>

    <p>Then reload Claude Code and verify:</p>

    <CodeBlock language="text">{`Run /mcp and confirm glyphs-mcp is listed.
Then call list_open_fonts and summarize the result.
If you see an error, quote it verbatim.`}</CodeBlock>
  </TabItem>
</Tabs>

## If it still fails

- Confirm the local endpoint is running: `http://127.0.0.1:9680/mcp/`
- In Glyphs, open **Edit → Glyphs MCP Server Status…**
- Re-run the client command above
- See [Troubleshooting](./troubleshooting.mdx)

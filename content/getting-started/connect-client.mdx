All clients connect to the same local endpoint:

```text
http://127.0.0.1:9680/mcp/
```

## Preferred connection method

- Prefer **direct URL** connections when the client supports it (simpler, fewer moving parts).
- Use a **proxy** like `mcp-remote` when a client requires it (this typically requires Node 20+).
- If tokens matter, pick a narrower **Tool Profile** first in **Edit → Glyphs MCP Server Status…** so your client sees fewer tools/schemas. (Guide: [Settings & tool profiles](./start-server.mdx#settings-tool-profiles).)

:::tip Startup order
If your client can’t connect, start Glyphs fresh, start the MCP server, then launch/reload your client.
:::

## Verifying the connection

If you’re unsure whether the integration is working end-to-end, do this:

1. Ask your client to run `tools/list` (or equivalent) and confirm it can see the Glyphs MCP tools.
2. Run a read-only call:

```text
Call list_open_fonts and tell me how many fonts are open.
If you see an error, quote it verbatim.
```

Tip: if tokens matter, set a narrower Tool Profile in **Edit → Glyphs MCP Server Status…** before connecting so the client sees fewer tools/schemas. (Guide: [Settings & tool profiles](./start-server.mdx#settings-tool-profiles).)

## Errors and troubleshooting

- If the endpoint works in a browser but your client fails: browsers hit discovery JSON, MCP clients need Streamable HTTP (SSE).
- If tool names/schemas look stale: reconnect/reload the client so it re-fetches `tools/list`.
- If `npx` fails in Claude Desktop/Cursor: pin Node 20+ via an explicit `env.PATH` (see the Claude Desktop tab above).
- For a full checklist: [Troubleshooting](./troubleshooting.mdx).

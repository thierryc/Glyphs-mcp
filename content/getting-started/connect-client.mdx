All clients connect to the same local endpoint:

```text
http://127.0.0.1:9680/mcp/
```

## Preferred connection method

- Prefer **direct URL** connections when the client supports it (simpler, fewer moving parts).
- Use a **proxy** like `mcp-remote` when a client requires it (this typically requires Node 20+).

:::tip Startup order
If your client canâ€™t connect, start Glyphs fresh, start the MCP server, then launch/reload your client.
:::

    <CodeBlock language="toml">{`[mcp_servers.glyphs-mcp-server]
url = "http://127.0.0.1:9680/mcp/"
enabled = true
startup_timeout_sec = 30
tool_timeout_sec = 120`}</CodeBlock>

    <p>
      Verify:
    </p>

  <TabItem value="claude" label="Claude Desktop">
    <img
      alt="Claude Desktop MCP registry screenshot"
      src={claudeSonnetImg}
    />

    <p>Claude Desktop reads configuration from:</p>

    <CodeBlock language="text">{`/Users/<userName>/Library/Application Support/Claude/claude_desktop_config.json`}</CodeBlock>

    <p>Append an MCP server entry and restart the app:</p>

    <CodeBlock language="json">{`{
  "mcpServers": {
    "glyphs-mcp-server": {
      "command": "npx",
      "args": ["mcp-remote", "http://127.0.0.1:9680/mcp/"],
      "env": {
        "PATH": "/Users/<userName>/.nvm/versions/node/v22.19.0/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin"
      }
    }
  }
}`}</CodeBlock>

    <p>
      Note: <code>mcp-remote</code> requires Node 20+. Claude Desktop does not inherit your login
      shell PATH, so set <code>env.PATH</code> explicitly.
    </p>

    <p>
      Verify:
    </p>

    <CodeBlock language="text">{`Call list_open_fonts and summarize the result.
If you see an error, quote it verbatim.`}</CodeBlock>
  </TabItem>
